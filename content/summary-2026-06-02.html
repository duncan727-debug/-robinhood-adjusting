The newsletter engine

3 audiences, 3 voices, one research pass

Daily brief skill researches once, writes three distinct briefs — homeowner, service provider, real-estate pro — with audience-specific "what it means" framing. Deduplicates against the prior four days. Sunday flips to a weekly trends report.

HubSpot-native delivery

Beehiiv retired May 2; sends now run on HubSpot's native email so subscribers, deals, and tasks live in one system. Calendly + IMAP bridge close the loop: replies post to deals within five minutes.

The CRM & outreach machine

Prospect → reply → task, fully automated

  • Daily consolidation 8:30am: research → hubspot_master_import.csv → API upload, no manual paste.
  • Outreach send: personalized voice, soft on first contact, intent ramps on follow-ups; A/B styles tracked.
  • IMAP bridge: scripts/imap_bridge.py runs every 5 min — bounces, replies, deal updates land in HubSpot automatically.
  • Contact-form fallback: after 3 bounces, escalates to website forms (~40% submit rate).
  • Email-type tasks → Gmail Drafts: Duncan reviews from Gmail, not HubSpot — actually gets done.

How we operate

Consent-first when stakes are high. Move first when they're not. Never fabricate a fact, a citation, or a number. Flag disagreement once, then execute. Write everything down — sessions reset, memory persists. Always offer the durable option, even when it costs more upfront.
