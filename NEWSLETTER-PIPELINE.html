🏗️ Current Pipeline Architecture

Tier 1: Research & Content (4-5 AM Daily)

4:00 AM — Daily Research Brief ├─ Input: News APIs, FL OIR, NOAA, county sources ├─ Deduplication: Against last 4 days └─ Output: /briefs/YYYY-MM-DD.md + .html (7K words) 4:30 AM — Content Creation (depends on brief) ├─ Extracts: 1-2 key angles from brief ├─ Generates: │ ├─ /content/YYYY-MM-DD/facebook.md │ ├─ /content/YYYY-MM-DD/instagram.md │ └─ /content/YYYY-MM-DD/blog.md └─ Output: Drafts (NOT auto-posted) 5:00 AM — Prospect Intelligence ├─ Input: /crm/organizations.csv ├─ Research: Decision makers, pain points └─ Output: /crm/intelligence/YYYY-MM-DD/ 6:00 AM — CRM Outreach (Mon-Fri) ├─ Input: Intel + /crm/templates.md (6 categories) ├─ Personalize: Email draft └─ Output: /crm/drafts/ (NOT auto-sent)

Tier 2: Website Deployment

5:30 AM — Git Sync + Build ├─ Commit: All briefs, content, intelligence, drafts to GitHub ├─ Trigger: Netlify webhook └─ Deploy: /PA-WEBSITE.html with latest 5 briefs (live in 2-5 min)

Tier 3: Newsletter Subscription

Website Form → Netlify Function → Beehiiv API ├─ Email validation (regex) ├─ POST /subscriptions └─ Result: Subscriber added to publication

Tier 4: Operations Monitoring

7:30 AM — Daily Ops Review (Mon-Fri) ├─ Assess: Brief quality, content volume, prospect extraction ├─ Identify: Bottlenecks & system health issues └─ Output: /ops-review/YYYY-MM-DD.md (auto-generated)

⚠️ Bottlenecks & Gaps

Gap 1: Brief → Prospect Extraction (High Priority)

Research is thorough but prospect naming/discovery is manual. Output: 5-7 qualified prospects per day identified but not extracted into action items.

Gap 2: Outreach Automation (High Priority)

Email drafts generated daily (6 AM) but never sent. Current: 5-7 emails waiting for manual send each day. Impact: Lost follow-up opportunities.

Gap 3: Social Media Integration (Medium Priority)

Facebook/Instagram drafts created but manually posted. Time tax on Duncan; inconsistent publishing schedule. No API automation.

Gap 4: Newsletter Distribution (Medium Priority)

Beehiiv form works; subscribers added. BUT briefs aren't sent as daily emails. Subscribers see website updates but don't receive email delivery.

Gap 5: HubSpot Sync (Medium Priority)

CSV export ready; Beehiiv integration configured (Zapier). But email migration blocked by Gmail flagging. No prospect lifecycle tracking yet.

Gap 6: Response Tracking Loop (Low Priority)

Response handler logs interactions but doesn't feed back into next day's intelligence. Personalization doesn't evolve with conversation history.
