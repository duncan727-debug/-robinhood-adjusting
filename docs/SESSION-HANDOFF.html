3. The upgrade itself

3a. What will happen when Duncan runs the upgrade

npm install -g openclaw@latest

This replaces the contents of /usr/local/lib/node_modules/openclaw/. Side effects to expect:

  • The openclaw-gateway process keeps running on the old binary until its LaunchAgent is reloaded — Duncan will need to launchctl kickstart -k gui/$UID/ai.openclaw.gateway (or full bootout/bootstrap) for the new code to take effect.
  • openclaw.json at ~/.openclaw/openclaw.json is NOT touched by npm install. Token, gateway port, plugin entries, agent defaults all preserved.
  • Config schema may have changed across 5 weeks of updates. If the new gateway rejects the old config, fall back to the most recent backup: ~/.openclaw/openclaw.json.bak (latest) or roll forward by running openclaw configure.
  • Control-UI URL will be the same: http://127.0.0.1:18789/#token=… with the same token (unless we rotate).
  • Bundled skills under /usr/local/lib/node_modules/openclaw/skills/ will refresh — workspace skill cache may rebuild on first use.
